# Запуск сайта (Django)

Сайт перенесён на Django. Заявки из форм сохраняются в БД и отображаются в админке.

## Требования

- Python 3.10+
- pip

## Быстрый старт

1. Клонировать репозиторий и перейти в каталог проекта.

2. Создать виртуальное окружение и установить зависимости:
   ```bash
   python -m venv venv
   venv\Scripts\activate   # Windows
   # source venv/bin/activate  # Linux/macOS
   pip install -r requirements.txt
   ```

3. Настроить окружение:
   ```bash
   copy .env.example .env   # Windows
   # cp .env.example .env  # Linux/macOS
   ```
   Отредактировать `.env`: задать `SECRET_KEY`, при необходимости `DEBUG`, `ALLOWED_HOSTS`, `DATABASE_URL` (для PostgreSQL).

4. Применить миграции и создать суперпользователя:
   ```bash
   python manage.py migrate
   python manage.py createsuperuser
   ```

5. Запуск:
   - **Разработка:** `python manage.py runserver` — сайт будет доступен по адресу http://127.0.0.1:8000/
   - **Продакшен:** см. [deploy/DEPLOY.md](deploy/DEPLOY.md) (gunicorn + Nginx).

6. Админка заявок: http://127.0.0.1:8000/admin/ (логин/пароль суперпользователя).

## Обновление сайта на VPS

После изменений локально:

1. Закоммитьте и отправьте изменения в `main`.
2. Подключитесь к серверу:
   ```bash
   ssh root@185.147.127.149
   ```
3. Запустите деплой:
   ```bash
   /usr/local/bin/deploy_site_fmr.sh
   ```

Скрипт сам выполнит: `git pull`, установку зависимостей, миграции, `collectstatic` и перезапуск сервиса.

Полезные команды проверки:

```bash
systemctl status site_fmr --no-pager
journalctl -u site_fmr -n 100 --no-pager
systemctl status nginx --no-pager
```

# Техническое задание на разработку корпоративного сайта

## 1. Общая информация о проекте

**Тип сайта:**
Корпоративный B2B-сайт компании в сфере автоматизации складской техники.

**Позиционирование компании:**
Компания **автоматизирует складскую технику под ключ**: выполняет дооснащение техники комплектом оборудования и контроллеров для перевода её в **беспилотный режим работы внутри помещения**.

**Что является продуктом компании:**
Не интеграция склада, не WMS/RMS и не комплексный ИТ-контур, а именно:

* установка контроллеров;
* установка сенсоров и компонентов беспилотности;
* дооснащение техники;
* настройка беспилотного режима;
* запуск и сопровождение решения на единице техники или парке техники.

**Что НЕ входит в позиционирование сайта:**

* интеграция с RMS/WMS;
* комплексная автоматизация склада;
* разработка складской ИТ-архитектуры;
* оркестрация всего склада;
* роль генерального подрядчика по автоматизации склада.

**Основная задача сайта:**
Показать, что компания умеет **переводить складскую технику в беспилотный режим под ключ**, быстро, безопасно и с понятным эффектом для клиента.

Итоговый сайт должен продавать следующую идею:

**Мы автоматизируем складскую технику под ключ: устанавливаем контроллеры и оборудование для перевода техники в беспилотный режим, запускаем решение на объекте и сопровождаем его.**

И отдельно важно, чтобы сайт **не создавал ложного впечатления**, что компания занимается:

* интеграцией WMS/RMS;
* комплексной автоматизацией склада;
* ИТ-платформой для управления всем объектом.

Если хочешь, я могу следующим сообщением собрать это в **короткое ТЗ на 1 страницу для подрядчика** или в **структуру меню сайта с уровнями разделов**.
